WP Travel Machine Documentation

A modern travel & hotel booking system for WordPress — trip packages, hotel rooms, a smart booking engine, an availability calendar, Gutenberg blocks, Elementor widgets and a full developer API.

Version 1.0.0 · Requires WordPress 6.0+ · PHP 7.4+

Overview

WP Travel Machine adds two content types — Trips and Hotels — plus a complete booking engine, payment gateways, an availability/inventory system, AJAX search & filtering, a wishlist, and reviews. Everything is rendered through a shared, style‑aware renderer so the output is identical whether it comes from a shortcode, a Gutenberg block or an Elementor widget.

TripsTour packages with itinerary, pricing tiers, FAQ & gallery. HotelsRooms, facilities, availability calendar & per‑night pricing. Blocks & WidgetsGutenberg + Elementor with full content & style controls. Developer APITemplate overrides, 25+ hooks, helpers & REST API.

Install

  1. Upload the wp-travel-machine folder to /wp-content/plugins/ (or install the ZIP from Plugins → Add New → Upload).
  2. Activate the plugin. On activation it creates the database tables and the required system pages.
  3. Go to Travel Machine → Settings and set your currency, payment method and display options.
  4. Optionally use the Demo Importer on the dashboard to load sample trips & hotels.

Quick Start

  1. Add a Trip or Hotel: Travel Machine → Trips → Add New Trip (or Hotels). Fill in the title, description, featured image, and the tabbed configuration panel.
  2. Set pricing: Trips use pricing tiers (Adult, Child…); Hotels use the Rooms tab (price per night, sale price, capacity).
  3. Publish. Your item appears on its archive (/trips/ or /hotels/) and single page automatically.
  4. Show a grid anywhere with a shortcode such as [wptm_trips count="6" columns="3"] or a block/widget.

Rooms

Rooms are stored in a dedicated table (not post meta). Each room has: name, type, description, price per night, sale price, max guests, bed type and size. The cheapest room sets the “from” price on listings; the booking form lets the guest pick a room type and recalculates room price × nights.

Hotel Facilities

Open a hotel → Facilities tab. Add facility groups (e.g. General, Wellness, Food & Drink); list one facility per line. Each facility automatically gets a matching premium icon (Wi‑Fi, Pool, Spa, Parking, Restaurant…), and facility names are mirrored into the wptm_hotel_facility taxonomy so the front‑end filter keeps working.

Icons: Names matching the standard set get a dedicated icon; any other name still receives a sensible icon by keyword. Extend the set with the wptm_hotel_facilities filter.

Availability & Calendar

Open a hotel → Availability tab to control which dates are bookable. Add periods with:

  • From / To dates
  • Rooms available (inventory)
  • Status — Available or Blocked (closed / fully booked)
  • Price/night override (optional, shown on the calendar)

With no rules, the hotel is available on every future date. The front‑end booking calendar then reflects everything:

Past dates → disabledBlocked → red stripedSold out → greyPrice override → shown per day

Search & Filters

The Search Form Builder

Under Travel Machine → Search Form you can enable/disable and reorder the search fields (Destination, Date, Guests, Budget, Duration, Activity, Difficulty, Trip Type). The form is output by [wptm_search_form] and progressively enhanced to filter results over AJAX.

🖼Screenshot — Search Form BuilderReplace with a screenshot of Travel Machine → Search Form.

The Filter Bar

Trip & Hotel archives (and shortcodes with filters="yes") show a premium filter bar — keyword, destination, activity/trip‑type/difficulty + price range (trips) or hotel‑type/facility + star rating (hotels), plus sorting. Filtering is live over AJAX and resets pagination.

Query‑var note: the search form namespaces its GET parameters under wptm_search[...] to avoid collisions with other plugins that register bare destination/activity query vars.

Customizing the query

Filter the underlying WP_Query args:

add_filter( 'wptm_trips_query_args', function( $args, $atts ) {
    $args['meta_key'] = '_wptm_price';
    return $args;
}, 10, 2 );

Pagination & Items per Page

Under Settings → Display:

  • Items Per Page — applies to both trip & hotel archives and the paginated shortcode listings.
  • Pagination Type — Numbered Pagination or an AJAX “Load More” button. Both keep filtering consistent (filtering resets to page 1).

Bookings & Checkout

Each Trip/Hotel single page shows a booking form (sidebar). Hotels select dates (range) + room type; trips pick a departure date (fixed‑length) or range. The price summary updates live. There is also a session cart + Checkout page for multi‑item orders.

Payments & Gateways

Configure under Settings → Stripe / PayPal / Manual Payment. A gateway only appears at checkout when it is enabled and fully configured (keys present), so customers never pick a method that can't work.

GatewayBehaviour
Bank Transfer (Manual)Creates the order, redirects to the confirmation page with your bank instructions (Settings → Manual Payment). Status: awaiting.
StripeRequires publishable + secret keys. Shows an inline Stripe Elements card field and is SCA / 3‑D Secure ready via PaymentIntents — stripe.confirmCardPayment() runs any issuer authentication, then the intent is verified server‑side (status, booking id and amount) before the booking is marked paid.
PayPalRequires client ID + secret (Sandbox or Live). Renders PayPal Smart Buttons; the order is created and captured server‑side via the REST API, and the captured amount is verified against the booking total before confirming.

For online gateways the booking row is created first (pending / unpaid), the gateway runs its charge or capture, and the customer is only redirected to the confirmation page once payment is verified on the server. A failed or abandoned payment leaves an unpaid booking and surfaces the error — a paid‑looking confirmation is never shown for free. Totals are always recomputed server‑side.

AJAX endpoints (all behind wptm_booking_nonce): wptm_create_booking, wptm_stripe_create_intent, wptm_stripe_confirm, wptm_paypal_create_order, wptm_paypal_capture_order. Online card capture covers the single‑item booking form; the multi‑item cart checkout accepts Bank Transfer only.
Customize the bank‑transfer text on the confirmation page with the wptm_bank_transfer_instructions filter, or register a custom gateway via wptm_payment_gateways / wptm_payment_methods.

My Bookings

The [wptm_my_bookings] page (/my-bookings/) lists the logged‑in user's orders with item, date, total, payment status and a link to each order's confirmation. After ordering, logged‑in customers are sent here; guests land on the single‑order confirmation page.

AI Features

WP Travel Machine ships a built‑in AI suite that helps you create trips faster, reply to customers, and guide visitors — all powered by the model you choose. Turn it on under Settings → AI Features.

Provider‑agnostic. Works with Anthropic (Claude), OpenAI, or any OpenAI‑compatible endpoint — Groq, Gemini, OpenRouter, Ollama and more. Every AI request is nonce‑protected, admin actions require a capability check, and public endpoints are rate‑limited per visitor (default 10/min, see the wptm_ai_rate_limit filter) to keep API costs predictable.
Updated 30 Jun 2026. The chat concierge and smart recommendations now return real, bookable trip & hotel cards (validated against your catalogue server‑side — the AI can't invent a trip) instead of plain text, and both now include hotels. The chat input is multi‑line and renders links, and the interface moved to the Inter typeface with a lighter, cleaner weight scale.

In the Trip editor, the Overview tab opens with an AI Trip Builder panel. Give it a destination, number of days, a style (adventure, luxury, family, honeymoon…) and a budget level, then click Generate Full Trip. AI Customer Replies NEW

Open any booking (Bookings → View) to reveal the Reply to Customer box. Click Draft with AI and the assistant writes a personalised, on‑brand email using the booking's context — customer name, item, dates, status, total and any special requests. Pick a tone (friendly, professional, apologetic, enthusiastic), optionally tell it what to cover, then edit and Send — delivered through the same branded email template as your confirmations — or Copy it elsewhere.

The reply box also works without AI as a plain "email this customer" tool. Sending depends on wp_mail() — verify delivery with Settings → Emails → Send test email.

Using WP Travel Machine in a Theme

You rarely need to touch templates — render anything with a shortcode, block or the helper functions. Common patterns:

Render a grid in a template file

// In any theme PHP file
echo do_shortcode( '[wptm_trips count="6" columns="3" filters="yes"]' );

Call the renderer directly

echo \WPTravelMachine\Blocks\Renderer::trips( [
    'count'   => 6,
    'columns' => 3,
    'accent'  => '#0ea372',
] );

Inject content around single pages (no template copy needed)

add_action( 'wptm_single_hotel_before_sidebar', function( $hotel_id ) {
    echo '<div class="my-promo">Free breakfast included!</div>';
} );

Load front‑end styles only where needed

The plugin enqueues public.css on all front‑end pages, plus the self‑hosted Inter font (bundled — no external request). Disable it with add_filter('wptm_enqueue_fonts','__return_false') if your theme provides its own fonts.

Icon System

WP Travel Machine ships a library of premium line icons used across cards, single pages and facilities. Render one anywhere:

echo wptm_icon( 'map-pin', [ 'size' => 18 ] );      // inline SVG
echo wptm_stars( 4 );                          // gold star row
echo wptm_facility_icon( 'Free WiFi' );        // name → icon

Add or override icons:

add_filter( 'wptm_icon_library', function( $icons ) {
    $icons['rocket'] = '<path d="..."/>'; // inner SVG of a 24×24 icon
    return $icons;
} );

add_filter( 'wptm_hotel_facilities', function( $map ) {
    $map['Rooftop Pool'] = 'waves';
    return $map;
} );

FAQ & Troubleshooting

I changed a setting but the front end looks the same.

It's almost always page caching (theme or a cache plugin/CDN). Hard‑refresh (Cmd/Ctrl+Shift+R) or clear your site cache.

Booking dates always show “Available”.

That's the default when a hotel has no availability rules. Add periods under the hotel's Availability tab to control inventory and block dates.

Stripe/PayPal don't appear at checkout.

A gateway only shows when enabled and its API keys are filled in (Settings → Payments).

My “All Trips/Hotels” page doesn't paginate.

Those pages use [wptm_trips] / [wptm_hotels], which paginate by default. Ensure the shortcode isn't set to paginate="no", then set Items Per Page in Settings → Display.

How do I translate strings?

All strings use the wp-travel-machine text domain — load a translation in /languages/ or via a translation plugin.
